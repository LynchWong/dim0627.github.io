gitのリポジトリ単位にproxyの設定をする

会社でもvimを使っててて、plugin管理はneobundle.vimを使ってるんだけど、GitHubで管理してる.vimrcを落としてくるときにちょっと手間取った。

まあ簡単に言うと社内から外に出るときはproxyを通るから、gitがうまいこと動いてくれなかったんですね。

.gitconfigでなんとかなる

じゃあこうしようってことで、~/.gitconfigをこんな感じにしたんですよ。

[https]
        proxy = http://proxy.com:8080

ひとまず解決したんだけど、そしたら社内のgitが使えなくなった。
